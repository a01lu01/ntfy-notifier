## 一、致命问题（程序无法正常运行）

### 1. 🔴 `pywin32` 未安装——托盘模块完全不可用

**验证结果**:
```
>>> import win32gui
ModuleNotFoundError: No module named 'win32gui'
```

**pip list 显示**: 安装了 `pywin32-ctypes`（0.2.3），但**没有** `pywin32`（306）。

**区别**:
- `pywin32-ctypes`：纯 ctypes 实现，只提供 `win32api` / `win32con` 等常量，**不提供** `win32gui.CreatePopupMenu`、`win32gui.CreateWindowEx` 等函数。
- `pywin32`：真正的 Win32 API 绑定，包含 `win32gui`、`win32ui` 等完整模块。

**后果**: `tray.py` 中所有 `import win32gui` 都会失败，被 `except` 静默吞掉 → 托盘窗口创建失败（`_g_hwnd = 0`）→ `Shell_NotifyIconW` 跳过 → **没有托盘图标，程序看起来什么都没启动**。

**修复**:
```bash
pip install pywin32
python -c "import win32gui; print('OK')"  # 验证
```

### 3. 🔴 tkinter 与 Win32 消息循环线程冲突（仍存在）

**文件**: `src/tray.py` + `src/ntfy_notifier.py`  
**问题未变**: 托盘的 `_pump()` 在子线程运行 Win32 消息循环，而 `on_settings` 回调（`_open_settings`）在该子线程中被调用，内部操作 Tk 控件（创建 `Toplevel` 窗口），违反 Tk 的线程安全要求。

**触发路径**:
1. 用户右键托盘图标 → `WM_RBUTTONUP` → `_proc()` → `TrackPopupMenu`
2. 用户点击"设置..." → `WM_COMMAND` (wparam=1) → `_proc()` → `_g_settings()` → `_open_settings()`
3. `_open_settings()` 在 Win32 线程中创建 `SettingsWindow` → 操作 Tk 控件 → **TclError 崩溃或死锁**

**修复**: 在 `_proc` 中将回调调度到 Tk 主线程：
```python
# tray.py _proc 中所有调用 _g_settings / _g_quit 的地方改为：
if _g_settings:
    _root_ref.after(0, _g_settings)   # 需要传入 _root 引用
if _g_quit:
    _root_ref.after(0, _g_quit)
```
或者更简单：在 `_open_settings` 本身包一层 `after`。

### 4. 🔴 `_mkicon()` 图标创建逻辑错误

**文件**: `src/tray.py` 第 30-56 行  
**问题详解**:

1. **GDI 绘图后没有正确构建 HICON**：代码先在 memory DC 上画圆，然后用 `windll.user32.CreateIcon()` 尝试创建图标。但 `CreateIcon` 的参数 `AND bitmask` 传了 `None`，在部分 Windows 版本上会返回 0。

2. **`DeleteObject(mdc)` 是错的**：Memory DC 应该用 `DeleteDC` 而不是 `DeleteObject`。调用 `DeleteObject(mdc)` 实际上什么也没删（GDI 会忽略对 DC 的 DeleteObject），导致 **GDI 句柄泄漏**。

3. **`SelectObject(mdc, br)` 后没恢复原 brush**：选中自定义 brush 后画圆，然后又选中 stock brush（`GetStockObject(5)` = WHITE_BRUSH）画内圆——这里原 brush 没被恢复，但旧 brush 引用已丢失，也是 GDI 泄漏。

4. **fallback `LoadImageW(0, 32512, 1, 16, 16, ...)` 是加载系统图标**：`32512` = `IDI_APPLICATION`，这个可以工作作为后备，但仅当 `CreateIcon` 失败时。

**后果**: 如果 `CreateIcon` 返回 0 且 `LoadImageW` 也失败（在无权限环境下），`_g_hicon = 0`，托盘图标显示为空白或系统默认。

**修复建议**: 简化图标创建，用 PIL 生成后转换：
```python
from PIL import Image, ImageDraw
img = Image.new('RGBA', (32, 32), (0, 0, 0, 0))
draw = ImageDraw.Draw(img)
draw.ellipse([2, 2, 29, 29], fill=(0, 120, 212, 255))  # #0078D4
img.save('icon.ico', format='ICO', sizes=[(16,16),(32,32)])
# 然后用 LoadImageW 加载 ico 文件
```

### 6. 🟠 `NID.szTip` 字段类型错误——tooltip 不会显示中文

**文件**: `src/tray.py` 第 138 行 / 第 162 行  
**代码**:
```python
("szTip", c_uint * 64),   # ← 用 c_uint 数组存字符串
...
tip = "ntfy-Notifier"
tip_arr = (c_uint * 64).from_buffer_copy(b"\x00" * 256)
for i, ch in enumerate(tip):
    tip_arr[i] = ord(ch)
n.szTip = tip_arr
```

**问题**: `Shell_NotifyIconW` 是 Unicode 版本，`szTip` 字段应为 `wchar_t[128]`（即 `c_wchar * 128`）。当前用 `c_uint * 64` 存 ASCII 码值，碰巧对纯 ASCII 有效，但：
- 不支持中文 tooltip
- 长度限制错误（应为 128 个 wchar，不是 64 个 uint）
- 不是标准用法，可能在某些 Windows 版本上显示乱码

**修复**:
```python
("szTip", c_wchar * 128),
# ...
n.szTip = "ntfy-Notifier"
```

### 7. 🟠 `_proc` 中 `WM_COMMAND` 消息判断错误

**文件**: `src/tray.py` 第 88-93 行  
**代码**:
```python
elif msg == 0x112 and wparam == ID_SETTINGS:  # WM_SYSCOMMAND
```

**问题**: `0x112` 是 `WM_SYSCOMMAND`，但 `TrackPopupMenu` 产生的是 `WM_COMMAND`（`0x111`），不是 `WM_SYSCOMMAND`。**右键菜单点击"设置"或"退出"时，消息是 `WM_COMMAND`，不是 `WM_SYSCOMMAND`，所以这些 case 永远不会命中。**

当前菜单之所以还能工作，是因为 `WM_COMMAND` 被 `DefWindowProc` 默认处理后，`lparam` 中的鼠标消息可能被二次处理——但这不可靠。

**修复**:
```python
elif msg == 0x111:  # WM_COMMAND
    if wparam == ID_SETTINGS:
        if _g_settings: _g_settings()
    elif wparam == ID_QUIT:
        if _g_quit: _g_quit()
```

### 10. 🟠 `_pump()` 中 `MSG` 结构体字段类型不匹配

**文件**: `src/tray.py` 第 212-220 行  
**问题**: `MSG` 结构体中 `hwnd`、`wParam`、`lParam` 用的是 `c_int`（32 位有符号），但在 64 位 Windows 上：
- `hwnd` 应为 `c_void_p`（指针宽度）
- `wParam` 应为 `c_ulonglong` / `c_void_p`（64 位）
- `lParam` 应为 `c_longlong` / `c_void_p`（64 位）

用 `c_int` 在 64 位系统上会导致 `PeekMessageW` 写入时**截断高位**，可能引起消息处理异常。

**修复**: 使用 `ctypes.wintypes.MSG` 内置结构体，或正确定义字段宽度。

## 五、启动失败根因分析（更新）

根据实际环境验证，**当前"无法正常启动"的最直接原因**：

1. **`pywin32` 没装**（#1）→ `import win32gui` 全部失败 → 托盘窗口创建被 `except: pass` 静默吞掉 → 没有托盘图标出现 → 用户以为程序没启动
2. **`winrt` 没装**（#2）→ 即使程序跑起来也收不到通知
3. **线程冲突**（#3）→ 即使托盘出来了，点"设置"也会崩

### 最快跑通路径

**先装依赖，不改代码**：
```bash

```
然后从项目根目录运行：
```bash
python -m src.ntfy_notifier
```

此时托盘应该能出来，点"设置"可能还会崩（线程问题 #3），但至少能看到托盘图标了。

**推荐一步到位方案**：用已安装的 `pystray` 重写 `tray.py`，一次性解决 #1/#3/#4/#6/#7/#10/#12/#14 共 8 个问题。`pystray` 内部正确处理了线程调度、图标创建、菜单弹出等所有细节。
